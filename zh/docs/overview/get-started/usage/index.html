最终使用

Jimmer的两种用法

  • 使用Spring Data Repository

  • 直接使用底层的SqlClient

使用Spring Data Repository​

定义Repository接口​

Jimmer整合了spring data，为Java用户和Kotlin用户各自提供了一个Repository基接口。

Javaorg.babyfish.jimmer.spring.repository.JRepository<E, ID>
Kotlinorg.babyfish.jimmer.spring.repository.KRepository<E, ID>

无论Java版还是Kotlin版，接口具备两个范型参数

  • E: 实体类型
  • ID: 实体ID类型

通过继承此接口，开发人员可以快速实现各种Repository，以BookRepository为例

  • Java
  • Kotlin
BookRepostory.java
package com.example.repository;

import com.example.model.Book;

import org.babyfish.jimmer.spring.repository.JRepository;

public interface BookRepository extends JRepository<Book, Long> {}
BookRepostory.kt
package com.example.repository

import com.example.model.Book

import org.babyfish.jimmer.spring.repository.KRepository

interface BookRepository : KRepository<Book, Long>
备注

和其他spring-data相同，定义好接口即可，无需用@org.springframework.stereotype.Repository修饰。Jimmer会自动实现这些接口并注册到Spring中。

但是有一个注意事项：

  • 默认情况下，自定义Repository所属的必须是SpringBoot主类所在包或其子包。

  • 否则，需要用@org.babyfish.jimmer.spring.repository.EnableJimmerRepositories修饰SpringBoot主类或其他Spring配置类，明确指定自定义Repository接口所在的包。

默认查询方法​

可以为自定义Repository接口添加默认方法，由开发人员自己实现复杂查询。

  • Java
  • Kotlin
BookRepository.java
public interface BookRepository extends JRepository<Book, Long> {

    BookTable table = Tables.BOOK_TABLE;

    default List<Book> find(
        @Nullable String name,
        @Nullable String storeName,
        
        @Nullable String authorName,
        
        @Nullable Fetcher<Book> fetcher 
    ) {
        AuthorTableEx author = TableExes.AUTHOR_TABLE_EX; ❶

        return sql()
            .createQuery(table)
            .whereIf(
                StringUtils.hasText(name),
                table.name().ilike(name)
            )
            .whereIf(
                StringUtils.hasText(storeName),
                table.store().name().ilike(storeName) 
            )
            .whereIf(
                StringUtils.hasText(authorName),
                table.id().in(
                    sql()
                        .createSubQuery(author) ❷
                        .where(
                            Predicate.or(
                                author.firstName().ilike(authorName),
                                author.lastName().ilike(authorName)
                            )
                        )
                        .select(
                            author.books().id()
                        )
                )
            )
            .orderBy(table.name())
            .orderBy(table.edition().desc())
            .select(table.fetch(fetcher)) 
            .execute();
    }
}
interface BookRepository : KRepository<BookStore, Long> {

    fun find(
        name: String? = null,
        storeName: String? = null,

        authorName: String? = null,

        fetcher: Fetcher<Book>? = null
    ): List<Book> =
        sql
            .createQuery(Book::class) {
                name?.takeIf { it.isNotEmpty() }?.let {
                    where(table.name ilike it)
                }
                storeName?.takeIf { it.isNotEmpty() }?.let {
                    table.store.name ilike it
                }
                authorName?.takeIf { it.isNotEmpty() }?.let {
                    where(
                        table.id valueIn subQuery(Author::class) { ❶❷
                            where(
                                or(
                                    table.firstName ilike it,
                                    table.lastName ilike it
                                )
                            )
                            select(table.books.id)
                        }
                    )
                }
                orderBy(table.name)
                orderBy(table.edition.desc())
                select(table.fetch(fetcher))
            }
            .execute()
}

  • ❶

    • Java: 子查询基于全局变量TableExes.AUTHOR_TABLE_EX，为了便于后续开发，取一个别名author。

    • Kotlin: subQuery接受一个lambda表达式。

      在子查询的lambda内部，table变量覆盖了外围查询的table变量。如果需要在子查询中引入父查询的表 (本例未演示，通常用于写相关子查询)，请使用parentTable变量

  • ❷ 创建子查询

毫无疑问，如果authorName参数没有指定，最终SQL不会包含子查询，没必要演示。

我们演示authorName参数被指定了情况 (为了简化讨论，不指定fetcher参数)。

  • Java
  • Kotlin
List<Book> books = bookRepository.find(null, null, "A", null);
val books = bookRepository.find(authorName = "M")

直接使用底层的SqlClient​

构建SqlClient​

SqlClient是Jimmer中最底层的API

Javaorg.babyfish.jimmer.sql.JSqlClient
Kotlinorg.babyfish.jimmer.sql.kt.KSqlClient

如果用开发人员为Spring注入SqlClient，则Jimmer的SpringBoot starter会自动创建一个

警告

对于Kotlin项目而言，必须在applicaiton.properties或application.yml中配置jimmer.language = kotlin。 因为，Jimmer根据jimmer.langage的设置决定自动创建JSqlClient或KSqlClient。

开发人员也可覆盖Jimmer自动创建的SqlClient，如下

  • Java
  • Kotlin
@Bean
public JSqlClient sqlClient() {
    return JSqlClient
        .newBuilder()
        .setDialect(new PostgresDialect())
        ...省略其他配置...
        .build();
}
@Bean
fun sqlClient(): KSqlClient =
    newKSqlClient {
        setDialect(new PostgresDialect())
        ...省略其他配置...
    }
信息

如果选择覆盖Jimmer默认提供的SqlClient，建议让其名称为sqlClient。

否则，就需要在SpringBoot Application或某个Spring配置类上使用@EnableJimmerRepositories(sqlClientRef = "mySqlClient")

使用SqlClient​

  • Java
  • Kotlin
@SpringBootApplicattion
public class App {
    
    public static void main(args: Array<String>) {
        ApplicationContext ctx = SpringApplication.run(App.class, args);
        JSqlClient sqlClient = ctx.getBean(JSqlClient.class);
        List<Tuple3<Book, Integer, Integer>> tuples = sqlClient
            .createQuery(table)
            .select(
                // 第一列，以Book为聚合根且形状受对象抓取器控制的数据结构
                table.fetch(
                    Fetchers.BOOK_FETCHER
                        .allScalarFields()
                        .authors(
                            Fetchers.AUTHOR_FETCHER
                                .allScalarFields()
                        )
                ),
                // 第二列：以Native SQL的方式嵌入的分析函数
                Expression.numeric().sql(
                    Integer.class,
                    "rank() over(order by %e desc)",
                    table.price()
                ),
                // 第三列：以Native SQL的方式嵌入的分析函数
                Expression.numeric().sql(
                    Integer.class,
                    "rank() over(partition by %e order by %e desc)",
                    new Expression[] { table.storeId(), table.price() }
                )
            )
            .execute();
        for (Tuple3<Book, Integer, Integer> tuple : tupes) {
            System.out.printf(
                "Book object: %o, Global rank: %d, Local rank: %d\n",
                tuple.get_1(),
                tuple.get_2(),
                tuple.get_3()
            );
        }
    }
}
@SpringBootApplicattion
class App

fun main(args: Array<String>) {
    val ctx = runApplication(App::class, *args)
    val sqlClient = ctx.getBean(KSqlClient::class.java)
    val tuples = sqlClient
        .createQuery(Author::class) {
            select(
                // 第一列，以Book为聚合根且形状受对象抓取器控制的数据结构
                table.fetchBy {
                    allScalarFields()
                    authors {
                        allScalarFields()
                    }
                },
                // 第二列：以Native SQL的方式嵌入的分析函数
                sql(Int::class, "rank() over(order by %e desc)") {
                    expression(table.price)
                },
                // 第三列：以Native SQL的方式嵌入的分析函数
                sql("rank() over(partition by %e order by %e desc)") {
                    expression(table.store.id)
                    expression(table.price)
                }
            )
        }
        .execute()
    for ((book, globalRank, localRank) in tuples) {
        println("Book object: $book, Global rank: $globalRank, Local Rank: $localRank")
    }
}
